# 吃药么 (Chi Yao Me) — 项目说明

## 这是什么

面向**中国老年用户**的 Windows 桌面端吃药提醒应用。家属(常为远程)负责配置,老人是实际使用者。

核心设计原则,任何改动都不能违背:

- **离线优先**:所有核心功能必须在无网络时正常工作。
- **无障碍**:大字号、高对比、超大点击区域;信息采用 **图标 + 颜色 + 文字三重编码**(不能只靠颜色区分,要考虑色弱用户)。
- 面向不熟悉电脑的老人,交互越简单越好;复杂配置放到家属用的设置页。

## 技术栈

- 后端:Python + **pywebview**
- 前端:原生 HTML / CSS / JS(无框架)
- 离线 OCR:**RapidOCR**(onnxruntime)
- 语音提醒:**pyttsx3**(离线)
- 数据存储:**SQLite**(`drugs.db`,药品库)+ JSON 文件(用户/日程数据,位于 `~/.chiyaome/`)
- 模糊中文药名匹配:**rapidfuzz**
- 后台:定时线程负责 winsound 闹铃 + 离线语音播报
- 开机自启:写 Windows 注册表(HKCU,无需管理员);托盘常驻用 **pystray**
- 打包:**PyInstaller** → Windows exe

## 已完成的功能

- **药品数据管线**:`fetch_openfda.py` 从 openFDA API 拉取约 74 种老年常用药;`build_drugs_db.py` 将其与**手写中文药卡**(用法、老年人警示、禁忌)合并生成 `drugs.db`。
- 药盒拍照识别:带**强制 5 秒免责门**,以及两个官方核验按钮(NMPA 国家药监局、摩熵)。
- 月历视图:用药日高亮,**集成在已有的「今日」标签里**(日/月切换,不是新开标签)。
- 用药疗程/时长跟踪。
- 药品库扩展:手动录入 + JSON 导入,放在自定义设置页。
- 应用内功能介绍页 `intro.html`;以及纯浏览器、带 JS mock 后端的交互式 demo `demo.html`。
- 周报:按"餐次"统计用药依从性,单元格按颜色编码。

## 关键架构决策与理由(光读代码看不出来)

- **数据源选 openFDA**:作为离线数据库来源,以预构建好的 `drugs.db` 随应用分发。评估过 NMPA、openFDA、摩熵数科,最终选 openFDA 并**手工补一层中文内容**。
- **手写中文药卡,而非逐字翻译美国标签**:这是**安全考量**——美国标签直译可能误导中国用户;用简洁、经过审视的中文药卡更稳妥。**不要**把这条改成自动翻译 US label。
- **免责门只作用于药盒拍照识别**,不作用于手动录入。
- **官方站点深链失败时**:复制药名到剪贴板,提示用户手动粘贴搜索。
- **药盒识别走独立的 `recognize_box()` 路径**:不能复用为处方行设计的 `parse_text()`;**国药准字批准文号正则是主匹配锚点**。

## 明确砍掉 / 不在范围内的东西

- **家属短信 / 微信通知(原需求 5)已明确放弃**,不要重新引入。原因:云短信需要监管审批(签名/模板审核),不可行;唯一免审批的短信路径是硬件 GSM 模块 + SIM 卡走 AT 指令——成本与复杂度不值得,故砍掉。

## 打包 / 构建注意事项(踩过的坑,配 CI 时务必遵守)

- **Python 版本用 3.12**(`py -3.12`)。Python 3.14 没有 onnxruntime 的 wheel。
- PyInstaller **必须显式包含 pywebview 运行时依赖**:`proxy_tools`、`bottle`、`pythonnet`、`pywin32`。
- `build.bat` **必须是 CRLF 行尾**;Linux 的 LF 会让 cmd.exe 崩溃。**不要用 `^` 续行**。
- `requirements.txt` 注释**必须用 `#`,不能用分号**(pip 会把 `;` 当作环境标记 environment marker 解析)。
- `build.bat` 里指向已删除文件的过期 `--add-data` 会导致构建报错;**构建开始时应自动清理 `build/`、`dist/`、`*.spec` 缓存**。

## 仓库 / 发布约定

- **不提交**:`build/`、`dist/`、`*.spec`、`__pycache__/`、`*.pyc`,以及任何本地用户数据(`~/.chiyaome/` 本就在仓库外)。
- `drugs.db` 倾向**不提交**,改为靠 `build_drugs_db.py` 重建,成品库作为 GitHub Release 附件分发。
- 正式版 exe 通过 **GitHub Releases** 分发(目标用户是老人和家属,不会自己从源码打包)。
- `demo.html` 适合用 **GitHub Pages** 托管,方便他人免安装在线试用。
- README 顶部需有**醒目的医疗免责声明**:本工具仅做提醒,不构成医疗建议,用药请遵医嘱。
- 数据来源需标注 openFDA。

## 开发节奏

迭代式开发,按批次推进;每次改动前先确认架构合理、必要时先出技术设计文档,再写代码。
